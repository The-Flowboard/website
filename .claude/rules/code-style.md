# Code Style

## PHP

- **Version:** PHP 7.4+ with MySQLi extension
- **DB connection:** Always use global `$conn` from `php/db_config.php` — never create a new connection
- **SQL:** Always use prepared statements with `?` placeholders — never concatenate SQL strings
- **Input validation:** `filter_var($email, FILTER_VALIDATE_EMAIL)` · `!empty()` + `trim()` on all required fields · `intval()` for numeric IDs
- **Output escaping:** `htmlspecialchars()` on all user data rendered to HTML
- **Error handling:** Log with `error_log()`, never echo errors to the browser
- **AJAX responses:** Always return `{"success": bool, "message": "..."}` JSON — optionally include `"data"` key
- **Admin endpoints:** Session check at the very top of every file in `admin/ajax/`
- **Public API endpoints:** Verify Bearer token `KjGgRa5qd8Yzs3Di1Ayew2Qksn8cGH2P9OYdqlwOxhY=` via `Authorization` header
- **Webhooks:** Always async via `WebhookQueue::enqueue()` — never call n8n directly from a request (5s timeout risk)
- **Timezone:** `America/Toronto` set in both PHP and MySQL session — do not change

## MySQL

- **DB:** `jmc_website` · charset `utf8mb4` · user `jmc_user`
- **IDs:** Always `intval()` before using in queries
- **New tables:** Update the schema section in CLAUDE.md and add migration SQL to `database/`
- **Indexes:** Add indexes on columns used in WHERE clauses (status, email, referral_source)

## JavaScript (Vanilla only)

- No frameworks — no React, Vue, Angular, jQuery (jQuery exists in admin only)
- AJAX: use `fetch()` with JSON body for form submissions
- Form validation: HTML5 attributes first, JS second for custom messages
- Toast notifications: use existing toast pattern in `contact.html` (do not introduce a library)
- Event listeners: prefer `addEventListener` over inline `onclick`
- No `console.log` left in production code

## CSS

- **No hardcoded hex values** — always use CSS variables defined in the `:root` block
- **Glass morphism pattern** (every card/panel):
  ```css
  background: rgba(255,255,255,0.05);
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  border: 1px solid rgba(255,255,255,0.1);
  border-radius: 16px;
  ```
- **Transition:** `all 0.4s cubic-bezier(0.4, 0, 0.2, 1)`
- **Breakpoints:** 1024px (nav collapse) · 768px (grid → single col) · 480px (container padding)
- **Fonts:** Sora (headings, `font-family: var(--font-display)`) · Nunito Sans (body, `var(--font-body)`)
- **Icons:** Font Awesome 6.4.0 CDN only — no SVG icons, no emoji in UI chrome

## HTML

- Semantic elements: `<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<nav>`
- Every public page must include GA4 tag `G-5HH2RHZLZ7`
- Every public page must link to Google Fonts (Sora + Nunito Sans) and Font Awesome CDN
- Open Graph meta tags on every page (`og:title`, `og:description`, `og:image`)
- `<meta name="description">` max 160 characters

## File Naming

- PHP handlers: `noun_action.php` (e.g. `contact_handler.php`, `process_assessment.php`)
- Admin AJAX: verb or noun describing action (e.g. `save_blog.php`, `get_data.php`)
- HTML pages: kebab-case (e.g. `privacy-policy.html`)
- Blog images: `blog_{TIMESTAMP}_{RANDOM_HEX}.{ext}` (generated by upload scripts)
